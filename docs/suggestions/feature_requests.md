# Ingnomia Community Feature Requests

Distilled from **4,693 Discord messages** in #suggestions (Dec 2017 – Sep 2022).
322 conversation threads analyzed, deduplicated, and organized into actionable feature requests.

## Priority Legend
- **P1** — High demand, frequently requested, high impact
- **P2** — Solid request, moderate demand or complexity
- **P3** — Nice-to-have, niche, or very complex

## Summary

**100 deduplicated feature requests** from 5 years of community input.

### Implementation Status
- **~15 features already implemented** (needs, skills, schedules, pasture caps, seasons, fluids, mechanisms, trading, military, sound, monsters, anatomy, stockpile filters, groves, job amounts)
- **~5 partially implemented** (happiness stub, diseases/wounds, automaton tiers)
- **~80 not yet implemented**

### Top 10 Most Impactful (by community demand + game impact)

1. **Needs/mood system** — needs exist but happiness is a stub. Most-discussed feature
2. **Event/notification log** — universally wanted, nothing in codebase
3. **Stockpile UX overhaul** — filters exist but UX is poor, no auto-accept or stacking
4. **Production limits on workshops** — amount/repeat exist but no "craft until N in stock"
5. **Resizable UI + HUD stock counters** — most-reacted, nothing implemented
6. **Designated movement zones** — nothing in codebase, critical for combat
7. ~~Animal population caps~~ — **ALREADY DONE** (`pasture.cpp:m_properties.max`)
8. ~~Skill progression~~ — **ALREADY DONE** (Fibonacci-based leveling in `canwork.cpp`)
9. **Battle recap + combat UI** — nothing implemented
10. **Modding support** — DB is moddable but no formal API
